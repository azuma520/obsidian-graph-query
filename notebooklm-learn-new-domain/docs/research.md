# 研究文件：AI 輔助學習方法 — 開發前資料蒐集

## 研究背景

外部靈感來源：網路流傳「用 NotebookLM 在 48 小時內搞定大學課程」的學習法（據稱斯坦福學生分享）。方法核心是三個精準提問，讓 AI 快速建立認知骨架。

研究目的：蒐集用戶知識庫中已有的學習方法論素材，評估是否能融合成一個完整的「AI 輔助學習」工作流或 skill。

---

## 一、外部素材：48 小時學習法

### 方法摘要

把課程材料（教科書、論文、講義）全部丟進 NotebookLM，然後問三個問題：

| 步驟 | 問題 | 在做什麼 |
|------|------|---------|
| Q1 | 這個領域專家最常共識的 5 個核心思考框架是什麼？ | 要心智模型，不要表面摘要 |
| Q2 | 專家在哪三個核心議題上有根本分歧？每一邊的論點和證據？ | 攤開爭議點、共識、未解難題 |
| Q3 | 生成 10 個能區分「真的懂」跟「只是背」的問題 | 自測 + 錯誤追問「為什麼錯？漏了什麼？」 |

### 優勢

- 直接逼 AI 壓縮成教授級心智模型，跳過表面資訊
- 分歧問題讓你看到領域前沿，大部分學生從來不知道的
- 自測 + 追問錯誤 = 刻意練習

### 不足

- 只有「由上而下」，缺「由下而上」的實作驗證
- 48 小時能建地圖，但地圖不等於走過那條路
- 能跟老師對話 ≠ 真正理解，碰到實際應用會露餡
- 完全沒有執行線（應用 → 行動 → 復盤 → 萃取）

## 二、知識庫內部素材

### 2.1 核心理論：YJango 學習觀

來源筆記：`2、文獻筆記/YJango-學習觀.md`

核心論點：
- 學習 = 用有限例子找出輸入→輸出的規律，壓縮成知識
- 記憶 ≠ 學習，理解才是學習
- 知識的目的 = 壓縮訊息
- 分而治之：大知識拆成小知識，各個擊破
- 二階知識：找出「知識與知識之間的關係」= 觸類旁通

學習原則（YJango 版）：
1. 明確任務輸入輸出
2. 將信息壓縮成知識
3. 例子重塑大腦連結
4. 二階知識拆分知識

### 2.2 知識四層級

來源筆記：`2、文獻筆記/如何快速理解一門知識.md`（道長的思維舖子）

| 層級 | 定義 | 舉例 |
|------|------|------|
| 原理 | 底層邏輯，通用法則 | 公理、定律、效應 |
| 模型 | 原理的結構化、可視化表達 | 方程、矩陣、系統 |
| 操作 | 模型的流程化步驟 | 方法、流程、清單 |
| 經驗 | 真實情境的執行反饋 | 案例、故事、數據 |

兩條學習路徑：
- **由上而下**：原理→模型→操作→經驗（先框架再細節）
- **由下而上**：經驗→操作→模型→原理（先實作再歸納）
- 結論：不管哪條路，實作才能出真知

學習步驟（道長版）：
1. 找出原理及邏輯
2. 找出能將原理結構化的模型
3. 找到能具體執行的步驟及佐證實例

用戶洞察筆記：「區分層級不是為了分類本身，而是為了定位、應用、管理與演化」

### 2.3 華羅庚 讀厚讀薄

來源筆記：`6、永久筆記/2、教育、學習方法/我認為華羅庚的把書讀厚、再把書讀薄是什麼意思.md`

- **讀厚** = 收集夠多的例子，廣泛涉獵
- **讀薄** = 透過內化、連結、歸納，將規律壓縮成知識

與 48 小時法的關係：48 小時法直接從「讀薄」開始，跳過了傳統的慢慢讀厚過程。

### 2.4 七個思考提示

來源筆記：`6、永久筆記/2、教育、學習方法/我的七個思考提示是什麼？.md`

原版（瓦基）：想起什麼 / 是什麼的一部份 / 跟什麼很像 / 跟什麼不同 / 為什麼重要 / 來自哪裡 / 真的是關於什麼

用戶改良版（七個思考提示）：
1. **這個知識我的理解為何？** — 回想及思考，產生洞見
2. **這個知識可以怎麼做結合？** — 與舊知識結合，形成系統
3. **與什麼很像？原因為何？** — 模式匹配，找共同底層邏輯
4. **與什麼差異？** — 釐清相似概念的相異之處
5. **重要性為何？** — 篩選對自己真正有幫助的知識
6. **讓我想起誰說的什麼話？** — 回想舊知識，加深記憶
7. **我有什麼不一樣的觀點？** — 找出自己的理解與洞見

> 後來演化為**連結六問**：聯想 / 相似 / 差異 / 應用 / 相關 / 來源

### 2.5 費曼學習法

來源筆記：`6、永久筆記/2、教育、學習方法/我的費曼學習法的步驟為何？.md`

用戶理解版本：
1. **輸入**：吸收訊息及案例
2. **內化**：變成自我認知及行動準據
3. **教學**：用知識指導行動、解決問題
4. **反饋**：透過行動修正知識

YJango 補充：費曼學習法核心不在「教別人」，而在**驗證知識** — 用已有知識解決新狀況，就驗證了學習。

### 2.6 框架化思維

來源筆記：`2、文獻筆記/小lin 如何快速學習一個領域 知識架構、結構化、框架化.md`

方法：
1. 參考他人的框架
2. 找關鍵字做心智導圖
3. 追問「為什麼？如何做？」帶著目的查找
4. 用時間線細分脈絡
5. 分析異常變化
6. 找出本質、過程、因果
7. 從流程思考如何優化

用戶洞察：「框架的目的是讓自己有思考的著力點」

### 2.7 麥肯錫三步法

來源筆記：`2、文獻筆記/強者思維-短時間成為行業精英的三個步驟.md`

1. 找到行業的 100 個關鍵詞
2. 讀 5 本書，透徹整個體系
3. 問 3 個專家，共同點就是最重要的東西

### 2.8 大腦與工具分工

來源筆記：`6、永久筆記/2、教育、學習方法/在學習的過程中，大腦與筆記的用途為何？.md`

- **筆記**：儲存及分類（外部系統）
- **大腦**：思考及內化（認知處理）
- 原因：大腦適合理解及推論，不適合記憶及儲存

### 2.9 AI 作為思考夥伴

來源筆記：`6、永久筆記/2、教育、學習方法/為什麼用AI 作為思考夥伴可以深化知識理解.md`

- AI 提供思考的「催化劑」和「鏡子」
- 解決了「不知道從何思考」的痛點
- 將筆記跟 AI 共享討論，AI 會一起討論內容

### 2.10 用戶已歸納的學習解決方案

來源筆記：`6、永久筆記/2、教育、學習方法/我如何解決我學習上的問題？.md`

1. 明確學習目標（興趣、目的、疑惑）
2. 為用而學（發展興趣、達成目的、解決疑惑）
3. 找出底層邏輯
4. 運用分而治之
5. 善用學習方法（費曼、卡片盒、HQ&A）
6. 活用思維架構，進行分類及連結

### 2.11 知識分類系統（現行）

來源文件：`圖查訊/types.md`

| 類型 | 定義 | 核心欄位 |
|------|------|---------|
| 概念 | 學到的理論、框架、原則（別人的想法） | source, domain |
| 經驗 | 實際做過的事、踩過的坑 | project, insight |
| 工具 | 研究過或正在用的工具、方法 | purpose, status |
| 洞察 | 自己想通的東西 | derived_from, validates/contradicts |

5. **經驗筆記 + applied_concepts** — 從知識到行動的記錄，48 小時法缺執行線

## 四、初步觀察（非結論）

1. **48 小時法是一個好的「入口協議」**，但不是完整的學習方法。它解決了「如何快速進入一個陌生領域」，但沒解決「如何把知識變成自己的」。

2. **用戶的知識體系已經有完整的拼圖**，包括：入口（框架化）→ 理解（思考提示）→ 連結（六問）→ 驗證（費曼）→ 外化（筆記系統）→ 分類（四層級 / 四類型）。缺的不是素材，是把它們串成一條流程。

3. **AI 的角色已經被定義過**：「催化劑和鏡子」。48 小時法用 AI 做框架提取，但用戶的體系裡 AI 還能做更多 — 思考夥伴、連結建議（graph-query / qmd）、結構化輔助。

4. **知識四層級和知識分類系統可以對齊**，但目前是兩套獨立的分類。如果融合，「知識屬性」欄位（原理/模型/操作/經驗）就是知識四層級的直接映射。

5. **兩條學習路徑都需要支持**：由上而下（48 小時法的場景）和由下而上（從實作中歸納的場景）。最終目標是它們能匯合。

## 七、AI 角色定義（共識）

### 7.1 設計定位

**思考引導器，不是筆記生成器。**

AI 的角色就兩個：
1. **提取** — 從材料中拉出術語、框架、分歧、測驗
2. **呈現** — 用不同形式呈現（文字、思維導圖、播客、測驗題）

用戶的角色始終是：**判斷、思考、回答**。

源自用戶早期洞察：AI 是思考的「催化劑」和「鏡子」。

### 7.2 工作流骨架 = 知識四層級

知識四層級本身就是工作流的骨架，每一層是下一層的地基：

| 知識層級 | 要做的事 | AI 工具 | 對應 48 小時法 |
|---------|---------|--------|---------------|
| 原理 | 術語、定義、基本邏輯、現象 | `notebooklm ask` | （缺失） |
| 模型 | 框架、結構化關係 | `notebooklm ask Q1` + `generate mind-map` | Q1 核心框架 |
| 操作 | 練習解決問題 | `notebooklm generate quiz` + 追問循環 | Q3 自測 |
| 經驗 | 實際去做 | **不做** | （缺失） |

48 小時法的不足：跳過原理層地基，直接從模型層開始；Q3 自測一次就停。

Q2（核心分歧）橫跨原理和模型層 — 可在兩個階段都使用。

### 7.3 產出兩層

| 層級 | 給誰 | 內容 |
|------|------|------|
| **方法論** | 所有人（含只有 NotebookLM 的用戶） | 知識四層級 + 對應的提問策略，手動按順序問就能用 |
| **自動化工作流** | 有 coding agent + 工具的 Obsidian 用戶 | agent 編排 notebooklm 指令 + 串接其他 skill |

核心是方法論，工具只是加速器。NotebookLM 免費、思維導圖免費、quiz 免費 — 不需要 coding agent 也能套用。

## 八、要做什麼（共識）

### 8.1 開發方向

以最小可用產品為目標：使用手冊 + 簡單介紹。方法論教材為未來延伸，現階段不做。

### 8.2 交付物

| 交付物 | 形式 | 給誰 | 優先級 |
|--------|------|------|--------|
| **提問模板** | 三個層級的具體提問句，可直接複製貼進 NotebookLM | 所有人 | 核心 |
| **使用手冊** | 怎麼用、什麼順序、為什麼有效 | 所有人 | 核心 |
| **SKILL.md** | 自動化編排層，調用 notebooklm + 其他 skill | 有 agent 的用戶 | 核心 |

### 8.3 工作流範圍

骨架 = 知識四層級的前三層：

| 層級 | 做什麼 | 工具 |
|------|--------|------|
| 原理 | 提取術語、定義、基本邏輯、現象 | `notebooklm ask` |
| 模型 | 提取框架 + 思維導圖 + 攤開分歧 | `notebooklm ask` + `generate mind-map` |
| 操作 | 生成測驗 + 追問循環 | `notebooklm generate quiz` + 多輪對話 |
| 經驗 | **不做** | — |

### 8.4 未來可延伸（現階段不做）

- 方法論教材（教學用途）
- 自動筆記產出（與工作流解耦後疊加）
- 串接 graph-query / qmd（Obsidian 用戶的進階功能）

## 九、不做清單（共識）

1. **不做經驗層** — 工作流到「知識準備好、結構化好」為止，實際應用是用戶的事
2. **不做額外工具整合** — 用現有工具能力開發，不引入新工具（除非確有必要）
3. **不幫做筆記** — 現階段工作流專注在引導思考，不自動產出筆記。寫筆記是之後可疊加的功能，與工作流解耦
4. **不綁死順序** — 步驟可跳、可挑，不強制序列執行

設計定位：**思考引導器，不是筆記生成器**。
