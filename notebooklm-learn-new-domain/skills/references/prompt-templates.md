# 提問模板：用 NotebookLM 學習陌生領域

> 把學習材料上傳到 NotebookLM 後，照以下提問句依序（或挑選）使用。
> 每個提問句可以直接複製貼上，不需修改。

## 上傳前：資料策劃檢查（3 分鐘）

> 學習效果的天花板不在提問，而在你上傳了什麼材料。花 3 分鐘做這個檢查，能避免後面的提問空轉。

**好的材料組合 = 三角形：全貌 + 權威 + 觀點**

| 角色 | 數量 | 做什麼用 | 怎麼找 |
|------|:----:|----------|--------|
| 全貌型 | 1-2 篇 | 讓你看見整個領域的鳥瞰圖 | Wikipedia、「XX 入門指南」、教科書導論 |
| 權威型 | 2-3 篇 | 提供準確的定義和細節 | 官方文檔、經典論文、技術規格 |
| 觀點型 | 1-2 篇 | 提供不同視角和爭議 | 評論文章、專家部落格、比較文、爭議報導 |

**快速檢查：**

> ✅ 有沒有一篇「告訴你全貌」的文章？（Wikipedia 是最簡單的選擇）
> ✅ 有沒有官方或權威來源？（教科書、官方文檔、經典論文）
> ✅ 有沒有「有人在吵」或「有人在比較」的文章？（評論、觀點文、比較文）
>
> 三個都有 → 學習效果最好。
> 只有官方文檔 → Q6 的分歧辨識會空轉，但替代方案比較（Q6-B）仍然有效。
> 只有科普 → Q4（底層邏輯）可能太淺，建議補一篇深度文章。

**找不到「觀點型」材料？** 試試 NotebookLM 的內建功能：點「Add source」→「Research」→ 輸入關鍵字，讓 NotebookLM 自動搜集相關資源。

## 原理層：打地基

> **目標：** 搞懂這個領域在講什麼 — 專有名詞、基本邏輯、重要現象。
>
> 這是所有理解的起點。不搞懂這些基礎，後面的框架和練習都是空中樓閣。

### Q1：核心術語

```
請列出這個領域最重要的 10-15 個專有名詞，每個詞給我：
1. 一句話定義（用日常用語，不要用其他專有名詞來解釋）
2. 為什麼這個詞重要（它在這個領域扮演什麼角色）
```

> 💡 **這個問題在幫你：** 建立這個領域的「基本詞彙表」。就像到一個新國家，先學會最常用的那幾個字。

### Q2：基本邏輯

```
這些核心概念之間的因果關係是什麼？
請用「A 導致 B」「A 是 B 的前提」「A 和 B 互相影響」這種方式說明，
讓我能看出它們是怎麼串在一起的。
```

> 💡 **這個問題在幫你：** 把散落的名詞連成一張網。知道概念之間怎麼串，比單獨記住每個詞重要得多。

### Q3：重要現象與效應

```
這個領域中，有哪些被廣泛認可的重要現象或效應？
每個現象請告訴我：
1. 它描述了什麼（用一句白話說）
2. 為什麼專家認為它重要
3. 日常生活中有沒有可以觀察到的例子
```

> 💡 **這個問題在幫你：** 從抽象概念拉回具體世界。現象是理論的具體表現，看得見、感受得到。

### Q4：底層邏輯

```
支撐這個領域的最底層邏輯是什麼？
也就是說，如果只能記住 2-3 條最基本的原則，
整個領域的大部分知識都能從這幾條原則推導出來——
那會是哪幾條？請用最白話的方式說明。
```

> 💡 **這個問題在幫你：** 找到這個領域的「根」。掌握底層邏輯，其他知識點就是樹枝和葉子，不用硬記也能推出來。

---

> **原理層完成檢查：** 做完這四個提問後，你應該能——
> - 看到這個領域的文章時，大部分術語你認得
> - 知道核心概念之間的關係（誰影響誰、誰是誰的前提）
> - 說出這個領域最重要的現象和底層原則
>
> 如果以上三點你都有感覺了，就可以進入下一層。

## 模型層：搭骨架

> **目標：** 找出專家怎麼「想」這個領域 — 他們用什麼框架思考、在哪些地方有分歧。
>
> 原理層讓你認得字，模型層讓你看見地圖。

> **銜接：** 如果你已完成原理層，你現在知道了術語和基本邏輯。接下來要問的是：專家們把這些知識整理成了什麼樣的思考架構？

### Q5：核心框架

```
這個領域中，專家最常使用的思考框架或模型有哪些？
請列出 3-5 個最重要的，每個框架告訴我：
1. 這個框架在解決什麼問題
2. 它的核心邏輯（用一兩句話說清楚）
3. 什麼情況下會用到它
```

> 💡 **這個問題在幫你：** 拿到專家的「思考工具箱」。框架就是專家用來快速分析問題的捷徑，有了這些你就不需要每次從零思考。

### Q6：核心分歧 或 替代方案比較（二選一）

> **Q6 有兩個版本，選適合你的：**
>
> - 你在學的東西**有人在吵**嗎？（學派之爭、政策辯論、價值觀衝突⋯）
>   → 用 **Q6-A**（核心分歧）— 了解這領域還沒定論的地方
>
> - 你在學的東西**有多種做法可以比較**嗎？（技術選型、工具比較、流程設計⋯）
>   → 用 **Q6-B**（替代方案比較）— 了解什麼情況該用什麼
>
> - 不確定？兩個都問，NotebookLM 會自己判斷哪個答得出來。

#### Q6-A：核心分歧

適合：學術領域、倫理爭議、政策辯論、AI 安全、商業策略⋯

```
在這個領域中，專家們在哪些核心議題上有根本性的分歧？
請列出 2-3 個最重要的爭議，每個告訴我：
1. 爭議的焦點是什麼
2. 正方的觀點和主要證據
3. 反方的觀點和主要證據
4. 目前學界比較傾向哪邊（如果有的話）
```

> 💡 **這個問題在幫你：** 看到這個領域的「前沿」和「裂縫」。知道專家在爭什麼，你就比大部分初學者看得更深。

#### Q6-B：替代方案比較

適合：技術文檔、框架選型、工具學習、流程設計、商業決策⋯

```
在這個領域中，有哪些地方存在不同的做法或替代方案？
請列出 2-3 個最重要的選擇，每個告訴我：
1. 有哪些替代方案（或不同的做法）
2. 各自的優缺點
3. 什麼情況下該選哪個
4. 目前業界比較傾向哪個（如果有的話）
```

> 💡 **這個問題在幫你：** 知道不同方案的能力邊界。「老闆問你選哪個」的時候，你能有理有據地回答。

### Q7：框架之間的關係（跟著 Q6 版本選）

> **Q7 要跟 Q6 配對：** Q6-A → Q7-A，Q6-B → Q7-B。兩個都問了就用 Q7-A。

#### Q7-A：搭配 Q6-A（核心分歧）使用

```
你剛才提到的這些框架和分歧，它們之間的關係是什麼？
哪些框架互補？哪些框架互斥？哪些分歧源自不同的框架假設？
請幫我理清它們的結構。
```

#### Q7-B：搭配 Q6-B（替代方案比較）使用

```
你剛才提到的這些框架和概念之間的關係是什麼？
哪些是互補的？哪些是有先後依賴的？
請幫我理清它們的結構，畫出一個完整的知識地圖。
```

> 💡 **這個問題在幫你：** 把多個框架從「平行清單」變成「立體結構」。理解框架之間的關係，才算真正掌握了這個領域的知識地圖。

### 🗺️ 生成思維導圖

完成以上三個提問後，用 NotebookLM 的思維導圖功能，把框架、分歧、概念之間的關係可視化：

**手動操作：** 在 NotebookLM 筆記本頁面，點選「生成」→「思維導圖」（Mind Map）。

> 💡 **為什麼要做這一步：** 文字描述的結構不容易記住，圖像化之後一眼就能看出哪些概念是核心、哪些是分支。這張圖就是你的領域地圖。

---

> **模型層完成檢查：** 做完這一層後，你應該能——
> - 說出這個領域最重要的 3-5 個思考框架
> - （Q6-A）知道專家之間最大的 2-3 個分歧是什麼；或（Q6-B）知道 2-3 個重要的替代方案及選型依據
> - 在腦中（或思維導圖上）有一張框架之間的關係圖
>
> 如果你已經開始有「我比較同意 X 的觀點」（Q6-A）或「我知道什麼情況該選什麼」（Q6-B）的感覺了，恭喜，你已經在形成自己的判斷。

## 操作層：練真功

> **目標：** 驗證你是真的懂了，還是只是「覺得自己懂了」。
>
> 原理層打地基，模型層搭骨架，操作層就是實際走一遍看看會不會倒。

> **銜接：** 原理和模型都看過了，接下來要測試你的理解。不是背誦測試 — 是逼你用剛學的知識去思考和判斷。

### 📝 生成測驗

先用 NotebookLM 生成一組測驗題：

**手動操作：** 在 NotebookLM 筆記本頁面，點選「生成」→「測驗」（Quiz）。

> 💡 **為什麼要做這一步：** 測驗能幫你找出「覺得懂」和「真的懂」之間的差距。答對的表示你真的理解了，答錯的就是你還需要補強的地方。

### Q8：追問（答錯時使用）

如果測驗中有答錯的題目，把錯誤的答案貼回 NotebookLM 追問：

```
我剛才回答了 [你的答案]，但正確答案是 [正確答案]。
請告訴我：
1. 我的思考哪裡出了問題
2. 我漏掉了什麼關鍵概念
3. 正確的推理過程應該是怎樣的
```

> 💡 **這個問題在幫你：** 錯誤是最好的學習機會。知道「為什麼錯」比「記住正確答案」重要一百倍。這個追問能幫你找到思考的漏洞。

### Q9：表態提問（跟著 Q6 版本選）

#### 如果你做了 Q6-A（分歧）：

挑一個模型層發現的分歧，逼自己表態：

```
關於 [某個爭議話題]，我比較傾向 [正方/反方] 的觀點。
請挑戰我的立場：
1. 我的立場最大的弱點是什麼？
2. 對方最有力的論點是什麼？
3. 有沒有我可能忽略的第三種觀點？
```

#### 如果你做了 Q6-B（替代方案）：

挑一個 Q6-B 的替代方案比較，做出你的選擇：

```
關於 [某個選型問題]，我在 [我的場景] 中會選擇 [方案 A/B]。
請挑戰我的選擇：
1. 這個選擇在什麼情況下會出問題？
2. 選另一個方案的人，他們最有力的理由是什麼？
3. 有沒有我可能忽略的第三種做法？
```

> 💡 **這個問題在幫你：** 逼你從「旁觀者」變成「參與者」。當你被迫站邊、被挑戰，你對這個領域的理解會比只是「讀過」深好幾層。

### 🎙️ 播客驗證（被動費曼法）

最後，用 NotebookLM 生成一段播客對話：

**手動操作：** 在 NotebookLM 筆記本頁面，點選「生成」→「音訊」（Audio Overview）。

聽的時候注意：**哪些地方讓你覺得「等等，這個我好像不太確定」？** 記下來，那就是你還需要回去補強的部分。

> 💡 **為什麼要做這一步：** 聽別人講你剛學的東西，是最輕鬆的驗證方式。聽到熟悉的部分你會點頭，聽到模糊的部分你會皺眉 — 皺眉的地方就是你的弱點。

---

> **操作層完成檢查：** 做完這一層後，你應該能——
> - 通過大部分測驗題（不只是猜對，是知道為什麼）
> - 對領域內的至少一個爭議（Q6-A）或選型問題（Q6-B）有自己的立場和理由
> - 聽播客時大部分內容聽得懂，能指出自己還不確定的地方
>
> 走到這裡，你已經從「完全不懂」變成了「能跟這個領域的人聊」的程度。

## 附錄：多來源比對（進階技巧）

> 這一節適合已經上傳了多份不同觀點材料的用戶。如果你只有一份材料，可以跳過。

### 為什麼多來源比對有效

不同來源的作者有不同的立場和強調重點。當你問同一個問題但指定不同來源，就能看到：
- 哪些東西所有來源都提到 → 這是核心共識
- 哪些東西只有某些來源提到 → 這可能是特定學派的觀點
- 哪些地方來源之間互相矛盾 → 這就是真正的分歧

這就是麥肯錫「問三個專家，共同點就是最重要的東西」的邏輯。

### 怎麼做

在 NotebookLM 裡，你可以在提問時勾選特定來源（點擊來源旁的核取方塊）。

**步驟：**

1. 取消全選，只勾選**來源 A**
2. 問：「這個領域最重要的框架是什麼？」
3. 記下回答
4. 取消來源 A，勾選**來源 B**
5. 問同樣的問題
6. 比對兩個回答，找出共同點和差異

### 適合比對的問題

模型層的 Q5（核心框架）和 Q6（核心分歧）特別適合多來源比對，因為不同作者對「什麼是最重要的框架」和「最大的爭議是什麼」往往有不同答案。

### 自動搜集材料（需要 notebooklm-py）

如果你使用 notebooklm-py，可以用研究模式自動搜集相關材料：

```
notebooklm source add-research "你要學的領域關鍵字"
```

這會讓 NotebookLM 自動搜尋並加入相關的網路資源，省去你手動找材料的時間。
