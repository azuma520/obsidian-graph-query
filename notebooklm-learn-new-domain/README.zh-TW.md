# NotebookLM — 學習新領域

一套學習工作流，幫助你對陌生領域建立結構化理解。

把學習材料上傳到 [Google NotebookLM](https://notebooklm.google.com/)，按順序跑一組提問句，就能從零開始建立一個領域的知識地圖。所有回答都基於**你的材料**，不是通用 AI 知識。

## 怎麼運作

工作流把學習拆成 3 層，每一層是下一層的地基：

| 層級 | 你在做什麼 | 你會拿到什麼 |
|------|-----------|-------------|
| **原理層**（Q1-Q4） | 提取術語、邏輯、現象、底層原則 | 詞彙表 + 因果網路 + 3 條核心原則 |
| **模型層**（Q5-Q7） | 找出專家框架、分歧或替代方案、知識地圖 | 3-5 個框架 + 一張結構圖 |
| **操作層**（Q8-Q9） | 測驗自己、追問錯誤、形成觀點 | 經過驗證的理解 + 你自己的判斷 |

> [!TIP]
> Q4（底層邏輯）在 4 次跨領域測試中，穩定產出 3 條記得住的原則，讓你能推導整個領域的知識。這是整套工作流中最強的單一產出。

## 兩種使用方式

### 手動版（人人可用）

不需要任何工具。打開[提問模板](skills/references/prompt-templates.md)，把提問句複製貼到 NotebookLM，照順序閱讀回答。

完整方法論說明請看[使用手冊](skills/references/user-guide.md)。

### 自動版（Claude Code 使用者）

安裝為 [Claude Code skill](https://docs.anthropic.com/en/docs/claude-code/skills)：

```bash
npx skills add <path-or-url>/notebooklm-learn-new-domain/skills
```

需要 [notebooklm-py](https://github.com/nicobailon/notebooklm-py)（>= 0.3.3）。Agent 會自動跑完全流程 — 資料策劃、提問選擇（包括 Q6 A/B 分支判斷）、生成物件、控制節奏。

## 提問總覽

| # | 提問 | 層級 | 在做什麼 |
|---|------|------|---------|
| Q1 | 核心術語 | 原理 | 建立術語表 |
| Q2 | 基本邏輯 | 原理 | 梳理因果關係 |
| Q3 | 重要現象 | 原理 | 辨識關鍵現象 |
| Q4 | 底層邏輯 | 原理 | 歸納底層原則 |
| Q5 | 核心框架 | 模型 | 提取思考框架 |
| Q6 | 核心分歧（A）或 替代方案比較（B） | 模型 | 辨識分歧 / 比較方案 |
| Q7 | 知識地圖（A/B） | 模型 | 建立知識結構 |
| Q8 | 追問 | 操作 | 針對錯誤追問 |
| Q9 | 表態（A/B） | 操作 | 形成自己的判斷 |

Q6 有兩個版本：**Q6-A**（核心分歧）適合學術/政策/倫理領域，**Q6-B**（替代方案比較）適合技術/工具/框架領域。Q7 和 Q9 跟著 Q6 連動。

## 資料策劃

學習效果的天花板不在提問，而在**你上傳了什麼材料**。目標是湊齊這個三角形：

| 角色 | 怎麼找 |
|------|--------|
| **全貌型** | Wikipedia、入門指南、教科書導論 |
| **權威型** | 官方文檔、經典論文、技術規格 |
| **觀點型** | 評論文章、觀點文、比較文 |

三種都有的話，學習效果最好。

## 適用領域

已在社會科學、技術規範、混合型（科技＋倫理＋商業）等不同性質的領域驗證過。

## 邊界

> [!NOTE]
> 這套工作流建立的是**知識地圖** — 走完之後你會知道這個領域的核心概念、關鍵框架、以及不同做法之間的取捨。下一步試著拿學到的東西去解決一個具體問題，知識在實踐中才會真正變成你的。
